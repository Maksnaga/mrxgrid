# Tutoriel — 7. Bulk actions et sélection

> Sélection multi-rows via checkboxes, Ctrl+A → Delete sur des
> centaines de milliers de cellules, modal de confirmation pour les
> actions destructives, export du subset sélectionné.

## 1. Activer la sélection

```vue
<MrxGrid
  …
  selectable
  selection-bar-compact
  @update:selection="onSelectionChange"
/>
```

- **`selectable`** — affiche la colonne checkbox à gauche
- **`selection-bar-compact`** — barre flottante en bas avec count
  + actions Edit/Copy/Paste/Delete (mode compact)
- **`@update:selection`** — émis à chaque changement de sélection,
  payload `SelectionModel`

```ts
export interface SelectionModel {
  allSelected: boolean
  selectedIds: Set<string>
  deselectedIds: Set<string>   // utilisé seulement en mode allSelected
}
```

Deux modes de sélection :

- **`allSelected: false`** — sélection explicite par row id. `selectedIds`
  contient les rowIds cochés.
- **`allSelected: true`** — Ctrl+A ou checkbox "tout" coché. `selectedIds`
  est vide, **`deselectedIds`** liste les exclusions. Cette
  optimisation permet de "sélectionner tout sauf 3" sans matérialiser
  500 000 ids.

<div className="callout callout--warn">

**⚠️ Piège — gérer les deux modes.** Si tu lis `selection.selectedIds`
sans vérifier `allSelected`, tu rates les Ctrl+A. Pattern correct :

```ts
function selectionCount(sel: SelectionModel, total: number): number {
  return sel.allSelected ? total - sel.deselectedIds.size : sel.selectedIds.size
}
```

</div>

## 2. Gérer la sélection côté composant

```ts
import type { SelectionModel } from '@mrx/grid'

const selectionIds = ref<number[]>([])

function onSelectionChange(payload: unknown): void {
  const sel = payload as SelectionModel | undefined
  if (!sel) {
    selectionIds.value = []
    return
  }
  if (sel.allSelected) {
    // Ctrl+A — on a tous les ids sauf les exclus. Pour des actions
    // bulk sur "tout sauf 2", on appelle l'API avec un endpoint
    // dédié `DELETE /products?except=42,87` plutôt que de
    // matérialiser le tableau de millions d'ids.
    selectionIds.value = ['ALL'] as any  // sentinelle
  } else {
    selectionIds.value = Array.from(sel.selectedIds, Number).filter(
      (n): n is number => !Number.isNaN(n),
    )
  }
}
```

## 3. Bulk delete avec confirmation

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { deleteProducts } from '@/api'
import DeleteConfirmModal from '@/components/DeleteConfirmModal.vue'

const deleteModalOpen = ref(false)
const deleteIds = ref<number[]>([])
const deleting = ref(false)

function askDelete(ids: number[]): void {
  if (ids.length === 0) return
  deleteIds.value = ids
  deleteModalOpen.value = true
}

async function confirmDelete(): Promise<void> {
  deleting.value = true
  const rowIds = deleteIds.value.map(String)
  try {
    // Row-level pending : grise les lignes ciblées + spinner pendant
    // la suppression. L'utilisateur voit visuellement ce qui disparaît.
    await pending.withRowPending(rowIds, () => deleteProducts(deleteIds.value))
    toasts.success(`${deleteIds.value.length} produit(s) supprimé(s)`)
    deleteModalOpen.value = false
    selectionIds.value = []
    // Silent refetch pour resync (les rows sont parties côté serveur).
    await list.refetch({ silent: true })
  } catch (err) {
    toasts.error((err as Error).message)
  } finally {
    deleting.value = false
  }
}
</script>

<template>
  …

  <DeleteConfirmModal
    v-model:open="deleteModalOpen"
    :count="deleteIds.length"
    :loading="deleting"
    @confirm="confirmDelete"
  />
</template>
```

## 4. La barre d'actions flottante

Quand des rows sont sélectionnées, une **action bar** apparaît en bas
de la grille (ou flotte en bas du viewport en compact mode) :

```
┌──────────────────────────────────────────────────────────────────┐
│ 12 rows selected   [✏️ Edit] [📤 Export] [🗑 Delete]  [Clear]   │
└──────────────────────────────────────────────────────────────────┘
```

Tu peux remplacer entièrement cette barre par ton propre composant :

```vue
<MrxGrid …>
  <template #action-bar="{ selection, clear }">
    <BulkActionBar
      :row-count="selectionIds.length"
      @export="onBulkExport"
      @edit-status="onBulkEditStatus"
      @delete="onBulkDelete"
      @clear="clear"
    />
  </template>
</MrxGrid>
```

Ou laisser celle par défaut et écouter les events :

```vue
<MrxGrid …
  @bulk-delete="onBulkDelete"
  @bulk-copy="onBulkCopy"
  @selection-edit="onSelectionEdit"
/>
```

## 5. Ctrl+A → Delete sur 100k+ cellules

Quand l'utilisateur fait Ctrl+A en mode cell selection (pas row
selection), il sélectionne **toutes les cellules** de toutes les rows
× toutes les colonnes éditables. À ces échelles (1M+ cellules), tu
**ne veux pas** que la lib émette un `cellEdit` par cellule — ce serait
1M de fan-out vers ton handler async = freeze.

La lib bascule automatiquement sur `bulk-cell-edit` au-delà de 1000
cellules :

```vue
<MrxGrid …
  @cell-edit="onCellEdit"
  @bulk-cell-edit="onBulkCellEdit"
/>
```

```ts
async function onBulkCellEdit(event: {
  changes: ReadonlyArray<{
    rowIndex: number
    field: string
    oldValue: unknown
    newValue: unknown
  }>
}): Promise<void> {
  // Group changes par field pour minimiser les API calls.
  // 1M cellules effacées sur 7 champs → 7 PATCH au lieu d'1M.
  const byField = new Map<string, { ids: number[]; value: unknown }>()
  for (const c of event.changes) {
    const row = list.rows.value[c.rowIndex]
    if (!row) continue
    let bucket = byField.get(c.field)
    if (!bucket) {
      bucket = { ids: [], value: c.newValue }
      byField.set(c.field, bucket)
    }
    bucket.ids.push(row.id)
  }

  list.refreshing.value = true
  try {
    const promises = Array.from(byField.entries()).map(([field, b]) =>
      updateProducts(b.ids, { [field]: b.value }),
    )
    await Promise.all(promises)
    toasts.success(`${event.changes.length.toLocaleString('fr-FR')} cellules effacées`)
    await list.refetch({ silent: true })
  } catch (err) {
    toasts.error((err as Error).message)
  } finally {
    list.refreshing.value = false
  }
}
```

<div className="callout callout--info">

**ℹ️ Threshold du bulk emit.** La lib émet `bulkCellEdit` au-delà de
**1000 changes** dans une seule opération (Ctrl+A → Delete, paste sur
range). Sous ce seuil, c'est `cellEdit` per-cell pour préserver les
semantics fines (validation, optimistic update, shimmer par cellule).
Si tu wires `cellEdit` mais pas `bulkCellEdit`, le clear de Ctrl+A
**ne fait rien** côté API (tes cellules sont effacées localement mais
pas persistées).

</div>

## 6. Export — respect de la sélection

L'API impérative `exportCsv()` détecte automatiquement la sélection :

```ts
const gridRef = ref<InstanceType<typeof MrxGrid> | null>(null)

function onBulkExport(): void {
  // Si sélection → exporte juste les rows sélectionnées (default)
  // Sinon → exporte les rows visibles
  gridRef.value?.exportCsv?.({ filename: 'catalogue-selection.csv' })
}

function onExportAll(): void {
  // Force le full dataset (toutes pages confondues)
  gridRef.value?.exportCsv?.({ filename: 'catalogue-full.csv', scope: 'all' })
}
```

Trois scopes possibles :

- **`'selection'`** — uniquement les rows sélectionnées (gère
  `allSelected` + `deselectedIds`)
- **`'visible'`** — rows actuellement rendues (post-filtre/tri,
  page courante)
- **`'all'`** — `props.rows` complet, indépendamment des
  filtres/tri/pagination

Par défaut, **`scope` est implicite** : `selection` s'il y a une
sélection, sinon `visible`.

<div className="callout callout--warn">

**⚠️ Piège — export sur 1M rows.** L'export sérialise tout en mémoire
(une string géante puis un Blob). À 1M rows × 50 cols, c'est
~300 MB → Chrome crash. **Toujours scopé par sélection** sur les gros
datasets. Si tu veux vraiment exporter 1M rows, génère le CSV
**côté serveur** et stream le download — pas via la lib.

</div>

## 7. Pattern composable récap

Voici la structure finale recommandée :

```ts
// vues/PimDashboard.vue
const list = useProductList()              // tuto 4 + 5
const pending = usePendingMutations()       // tuto 6
const toasts = useToaster()
const gridRef = ref<InstanceType<typeof MrxGrid> | null>(null)

// Sélection
const selectionIds = ref<number[]>([])
function onSelectionChange(p: unknown) { /* … */ }

// Bulk delete avec modal
const deleteModalOpen = ref(false)
const deleteIds = ref<number[]>([])
function askDelete(ids: number[]) { /* … */ }
async function confirmDelete() { /* … */ }

// Cell-edit & bulk-cell-edit
async function onCellEdit(e: CellEditEvent) { /* … */ }
async function onBulkCellEdit(e: { changes: … }) { /* … */ }
```

Et le template :

```vue
<template>
  <MrxGrid
    ref="gridRef"
    :columns="adeoColumns"
    :rows="list.rows.value"
    :total-count="list.total.value"
    :row-id="(row) => String(row.id)"

    :loading="list.loading.value"
    :refreshing="list.refreshing.value"
    :error="list.error.value"

    :pending-cells="pending.pendingCells.value"
    :pending-row-ids="pending.pendingRowIds.value"

    selectable
    sort-mode="server"
    filter-mode="server"
    :sort="list.sort.value"
    v-model:filter-model="list.filterModel.value"

    :pagination="{ pageSize: list.pageSize.value, pageSizeOptions: [25, 50, 100, 200] }"
    :height="640"

    @page-change="(e) => { list.page.value = Math.max(0, e.page - 1); list.pageSize.value = e.pageSize }"
    @sort-change="list.onSortChange"
    @filter-change="list.onFilterChange"
    @cell-edit="onCellEdit"
    @bulk-cell-edit="onBulkCellEdit"
    @update:selection="onSelectionChange"
  />
  <DeleteConfirmModal
    v-model:open="deleteModalOpen"
    :count="deleteIds.length"
    @confirm="confirmDelete"
  />
</template>
```

2. **Click Export depuis la barre** → CSV téléchargé avec 3 rows
3. **Click Delete → modal de confirmation** → confirmer → rows
   disparaissent, refetch silencieux
4. **Ctrl+A en cell mode → Delete** → `bulkCellEdit` émis,

   squelette plein écran (sauf le `refetch` explicite)
