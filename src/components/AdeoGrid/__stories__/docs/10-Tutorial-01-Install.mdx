# Tutoriel — 1. Installation et premier rendu

> On part de zéro. À la fin de cette page tu as un projet Vue 3 + Vite +
> TypeScript qui affiche un grid `AdeoGrid` vide à l'écran.

## Pré-requis

- **Node** ≥ 20.19 ou ≥ 22.12
- **Vue** 3.5+ (on suppose Composition API + `<script setup>` connus)
- **Vite** 7 ou 8
- **TypeScript** 5.9+ (strict recommandé)

Si tu connais pas Vue 3 Composition API, va lire
[la doc officielle](https://vuejs.org/guide/extras/composition-api-faq.html)
en parallèle — ce tuto ne ré-explique pas `ref` / `computed` / `watch`.

## 1. Scaffold du projet

```bash
npm create vite@latest mon-projet -- --template vue-ts
cd mon-projet
npm install
```

## 2. Installer adeo-grid + Mozaic

```bash
npm install @adeo/grid @mozaic-ds/vue @mozaic-ds/icons-vue @mozaic-ds/styles
```

`@mozaic-ds/styles` est obligatoire — c'est de là que viennent les
tokens (couleurs, spacings) que `AdeoGrid` consomme via CSS variables.

<div className="callout callout--warn">

**⚠️ Piège.** Si tu installes `@mozaic-ds/vue` sans les styles, le grid
s'affiche mais avec un look cassé : pas de couleurs, padding aléatoire,
icons sans alignement. La règle : **toujours les 4 packages ensemble**.

</div>

## 3. Importer les styles dans `main.ts`

```ts
// src/main.ts
import { createApp } from 'vue'
import App from './App.vue'

// Mozaic — tokens + composants
import '@mozaic-ds/styles/scss/main.scss'

// adeo-grid — styles internes du grid
import '@adeo/grid/styles.scss'

createApp(App).mount('#app')
```

L'ordre compte : Mozaic d'abord (les CSS vars sont définies), puis
adeo-grid qui les consomme. Inverse l'ordre et le grid utilise des
fallbacks au lieu des couleurs Adeo.

## 4. Premier composant

```vue
<!-- src/App.vue -->
<script setup lang="ts">
import { AdeoGrid, type ColumnDef } from '@adeo/grid'

const columns: ColumnDef[] = []
const rows: unknown[] = []
</script>

<template>
  <AdeoGrid :columns="columns" :rows="rows" :height="640" />
</template>
```

`npm run dev` — tu dois voir un grid vide avec un message "Le catalogue
est vide" et la toolbar par défaut (fullscreen, settings, etc.).

<div className="callout callout--info">

**ℹ️ Note.** `:height="640"` est obligatoire si tu veux que le grid ait
une taille fixe. Sans ça il prend la hauteur de son contenu — un grid
vide fait 0 px de haut. Tu peux aussi mettre `:fullscreen="true"` mais
c'est l'utilisateur qui devrait toggle ça via la toolbar, pas toi en
prop initial.

</div>

## 5. Vérifier la console

Ouvre la console navigateur. Tu ne dois voir **aucun** warning Vue ou
adeo-grid. Si tu en vois :

- `Failed to resolve component: MTag` → tu as oublié d'installer
  `@mozaic-ds/vue`.
- `Cannot read property 'value' of undefined` sur un computed Mozaic
  → tu as oublié `@mozaic-ds/styles` ou tu l'as importé après adeo-grid.
- `[Vue warn]: Invalid prop: type check failed for prop "columns"` →
  ta variable `columns` n'est pas un array (revérifie le `const`).
