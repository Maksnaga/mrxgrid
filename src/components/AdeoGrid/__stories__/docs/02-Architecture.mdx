## Flot de données

Le pipeline est **unidirectionnel** : `props.rows` traverse une chaîne
d'opérations pures et arrive aux cellules sous forme de slice virtualisée.

<div className="ascii">
{`props.rows  (input — N rows arbitraires)
     │
     ▼
┌────────────────┐
│ useSorting     │  multi-column sort stack
│  (engine.sort) │  → stable sort, comparator par colonne
└────────┬───────┘
         ▼
┌────────────────┐
│ useFiltering   │  ColumnDef.filter (legacy) + FilterModel (builder)
│ engine.filter  │  AND between conditions, OR within set ops
└────────┬───────┘
         ▼
┌────────────────┐
│ useGrouping    │  group fields → flat list with __mrx-prefixed metadata
│ engine.group   │  (group rows + data rows in one Array)
└────────┬───────┘
         ▼
┌────────────────┐
│ usePagination  │  page slicing OR infinite scroll markers
│ engine.paging  │
└────────┬───────┘
         ▼
┌────────────────┐
│ useVirtualGrid │  visible row range + visible column range
│ (vScroll +     │  → DOM minimaliste : 80 lignes max × 30 colonnes max
│  vColumns)     │
└────────┬───────┘
         ▼
   <MrxGridRow> × visibleRange
       <MrxGridCell> × visibleCols`}
</div>

## Conventions de design

<table>
  <thead>
    <tr><th>Choix</th><th>Pourquoi</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>State central via <code>provide/inject</code></td>
      <td>Évite le prop-drilling sur 5 niveaux ; chaque feature lit/écrit la même source.</td>
    </tr>
    <tr>
      <td>Métadonnées <code>__mrx</code></td>
      <td>Préfixées pour ne jamais collisionner avec les champs utilisateur (groupement, expansion).</td>
    </tr>
    <tr>
      <td>Pas de transform pour le scroll</td>
      <td><code>translateY</code> crée un containing block qui casse <code>position: sticky</code> sur les colonnes pinned. On utilise un spacer top-height à la place.</td>
    </tr>
    <tr>
      <td>Cap visible à 80 lignes</td>
      <td>Au-delà, le rendu d'une frame coûte plus que le scroll lui-même.</td>
    </tr>
    <tr>
      <td>RowHeight reactif (Ref ou number)</td>
      <td>La densité change la hauteur de ligne sans recréer le scroll machine.</td>
    </tr>
    <tr>
      <td>Style scoped systématique</td>
      <td>Isolation Vue scoped CSS + classes BEM pour les overrides via <code>:deep()</code>.</td>
    </tr>
  </tbody>
</table>
