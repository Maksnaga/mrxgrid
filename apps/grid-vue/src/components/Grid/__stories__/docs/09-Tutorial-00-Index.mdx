# Tutoriel — Index

> Un parcours pas-à-pas du minimum vital au scénario "vrai PIM avec
> API serveur, sélection multi, bulk delete, export". 8 chapitres
> indépendants — tu peux sauter celui qui te correspond pas.

## Pour qui

Devs Vue 3 (Composition API + `<script setup>` connus) qui découvrent
adeo-grid et veulent intégrer un grid dans leur projet sans relire
3000 lignes de source.

## Le scénario fil rouge

Du tuto 3 au tuto 7, on construit progressivement la même appli : un
gestionnaire de catalogue PIM Adeo (papier peint INSPIRE). On ajoute
une feature à chaque étape :

```
tuto 3   ► JSON statique, 58 cols, renderers Yes/No, formatters kg/cm
tuto 4   ► + API REST paginée, états loading/error/empty
tuto 5   ► + tri & filtre serveur, recherche debouncée
tuto 6   ► + édition de cellule avec pending shimmer + persistance
tuto 7   ► + sélection multi-row, bulk delete, export sélection, Ctrl+A
```

À la fin du tuto 7, tu as une appli prod-ready qui ressemble à la
demo `Adeo PIM` du Storybook.

## Conventions de cette doc

- **Encarts ⚠️ Piège** — bugs / surprises qu'on a vécus en interne.
  Lis-les, ils valent leur poids en heures de debug évitées.
- **Encarts ℹ️ Note** — clarifications "pourquoi on fait comme ça".
- **TypeScript strict** dans tous les exemples. Les snippets sont
  copy-pasteables tels quels (modulo les imports qu'on raccourcit
  parfois pour la lisibilité).

## Ce qui n'est PAS couvert

- **Tree mode** (parent / enfant hiérarchique) — voir
  [doc Architecture](?path=/docs/documentation-3-architecture--docs)
  pour l'API actuelle.
- **Formula engine** (`=SUM(A1:A10)` dans les cellules) — feature
  spécialisée, doc dédiée à venir.
- **Server grouping** — fonctionne mais demande un contrat API
  spécifique. À documenter en V2.
- **Plugins** — voir le chapitre `Plugins` de la doc API.
