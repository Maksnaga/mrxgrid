# Quick Start

> Construire ton premier `<AdeoGrid />` en 5 minutes — de l'install à un grid avec tri, filtres et virtualisation.

## Pré-requis

- **Node** ≥ 20.19 ou ≥ 22.12
- **Vue** 3.5+ (Composition API)
- **Vite** 7 ou 8 (Rolldown supporté)
- **TypeScript** 5.9+ (recommandé en strict mode)

## 1. Install

```bash
npm install @mozaic-ds/vue @mozaic-ds/icons-vue @mozaic-ds/styles @mozaic-ds/tokens
```

Le grid est livré avec le projet — copie le dossier `src/components/AdeoGrid` ou
publie-le en package interne.

## 2. Charger le thème Mozaic

Dans ton `main.ts` :

```ts
import { createApp } from 'vue'
// Choisis ton thème : '@mozaic-ds/tokens/theme' (Leroy Merlin par défaut),
// '/adeo/theme', '/bricocenter/theme' ou '/mbrand/theme'
import '@mozaic-ds/tokens/theme'
import '@mozaic-ds/vue/style.css'
import App from './App.vue'

createApp(App).mount('#app')
```

## 3. Premier grid

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { AdeoGrid, type ColumnDef } from '@/components/AdeoGrid'

const columns: ColumnDef[] = [
  { field: 'sku',   headerName: 'Référence', width: '120px', pinned: 'start' },
  { field: 'name',  headerName: 'Produit',   width: '260px', sortable: true, editable: true },
  { field: 'price', headerName: 'Prix',      width: '110px', sortable: true,
    valueFormatter: (v) => typeof v === 'number' ? `${v.toFixed(2)} €` : '' },
  { field: 'stock', headerName: 'Stock',     width: '100px', sortable: true, editable: true,
    cellEditor: 'number' },
]

const rows = ref([
  { sku: 'LM-001', name: 'Perceuse 18V Lexman', price: 79.9, stock: 42 },
  { sku: 'LM-002', name: 'Tondeuse électrique 1200W', price: 199, stock: 8 },
  { sku: 'LM-003', name: 'Mitigeur évier inox', price: 49, stock: 120 },
])
</script>

<template>
  <AdeoGrid :columns="columns" :rows="rows" />
</template>
```

Tu obtiens : un grid stylisé Mozaic, colonnes triables au click, cellules éditables
au double-click ou F2, ligne `Référence` toujours visible (pinned start).

## 4. Activer la virtualisation pour 100k lignes

Trois props suffisent : `virtual-scroll`, `container-height`, `row-id`.

```vue
<AdeoGrid
  :columns="columns"
  :rows="bigRows"
  virtual-scroll
  :container-height="600"
  :row-id="(row) => String(row.sku)"
/>
```

<div className="callout callout--info">
  💡 <code>row-id</code> est <strong>fortement recommandé</strong> dès qu'on dépasse
  quelques centaines de lignes — c'est ce qui permet à la sélection / aux formules /
  à l'expansion de survivre à un re-tri ou un lazy-load.
</div>

## 5. Pagination

```vue
<AdeoGrid
  :columns="columns"
  :rows="rows"
  :pagination="{ pageSizeOptions: [10, 25, 50, 100], defaultPageSize: 25 }"
  @page-change="onPage"
/>
```

L'évent `pageChange` remonte `{ page, pageSize, startIndex, endIndex }`.

## 6. Persistance du layout

Avec une clé stable, le grid sauve auto largeur / ordre / pin / sort / filtres
en `localStorage` et les restaure au reload :

```vue
<AdeoGrid :columns="columns" :rows="rows" persist-key="catalogue-v1" />
```

## 7. Brancher le toolbar et les drawers

```vue
<script setup lang="ts">
import { ref, computed } from 'vue'
import {
  AdeoGrid,
  AdeoGridToolbar,
  AdeoTableMenuDrawer,
  AdeoGroupingDrawer,
  AdeoGridFilterDrawer,
} from '@/components/AdeoGrid'

const settingsOpen = ref(false)
const groupingOpen = ref(false)
const filtersOpen  = ref(false)
const fullscreen   = ref(false)
</script>

<template>
  <AdeoGrid :columns="columns" :rows="rows" :fullscreen="fullscreen">
    <template #toolbar>
      <AdeoGridToolbar
        show-fullscreen show-settings show-group show-filters
        :fullscreen="fullscreen"
        @toggle-fullscreen="fullscreen = !fullscreen"
        @settings="settingsOpen = !settingsOpen"
        @group="groupingOpen = !groupingOpen"
        @filters="filtersOpen = !filtersOpen"
      />
    </template>
  </AdeoGrid>

  <AdeoTableMenuDrawer v-model:open="settingsOpen" :columns="columns" ... />
  <AdeoGroupingDrawer  v-model:open="groupingOpen"  :columns="columns" ... />
  <AdeoGridFilterDrawer v-model:open="filtersOpen"  :columns="columns" ... />
</template>
```

<div className="callout">
  ⚠️ Quand <code>:fullscreen</code> est activé, le grid devient
  <code>position: fixed; inset: 0</code>. Pour que la toolbar reste visible, place-la
  via le slot <code>#toolbar</code> du grid (et non en sœur du grid).
</div>

## Erreurs fréquentes

### 1. La dernière colonne ne remplit pas l'espace
La dernière colonne unpinned reçoit automatiquement <code>flex: 1 1 auto</code> dès que
la somme des colonnes est inférieure à la largeur du wrapper, <em>sauf</em> en mode
<code>:virtual-columns</code> (la slice change pendant le scroll).

### 2. Le grid déborde sa carte parente
Sans <code>:virtual-scroll</code> ni <code>:pagination</code>, le wrapper grandit naturellement
avec le contenu. Mets-le dans un parent <code>display: flex; flex-direction: column</code>
avec une hauteur définie ; le wrapper interne prend le reste via
<code>flex: 1; min-height: 0</code>.

### 3. Sort par click ne marche pas sur ma colonne
Vérifie que <code>sortable</code> n'est pas explicitement <code>false</code>. Le défaut est
<code>true</code>. Tu peux opter pour Shift+click (multi-tri) ou utiliser
<code>sortComparator</code> pour un ordre custom (grades énergie, locale FR, etc.).
